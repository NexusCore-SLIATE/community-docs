# 👑 Track Leads Management Playbook

Welcome to the leadership tier of NexusCore-SLIATE. As a Track Lead, your job is less about writing all the code yourself and more about engineering a sustainable environment where your team members can learn, execute, and collaborate safely.

---

## 📋 Core Responsibilities

### 1. Issue Architecture & Scoping
* Never let a member start coding without a linked item on the **NexusCore Master Delivery** project board.
* Break large project goals down into digestible, bite-sized issues. If a junior cannot complete a task within 3 to 5 days, the issue is too broad—break it down further.
* Label issues clearly using standard tags (`good first issue`, `bug`, `track: frontend`).

### 2. The 48-Hour Code Review SLA
* When a member opens a Pull Request (PR), aim to review it within **48 hours**.
* Look for clean folder structures, adherence to naming conventions (like `web-` or `api-` prefixes), and make sure no hardcoded API keys or passwords are naked in the commits.
* **Be an educator:** If code can be improved, explain *why* gently using inline PR comments. Never just close a PR with "this is wrong."
